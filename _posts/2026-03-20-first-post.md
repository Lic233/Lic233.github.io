# 基于 GitHub Pages 与 Jekyll 的博客网站搭建

实验内容与步骤：<br><br>
**GitHub Pages** 是 GitHub 提供的一项静态网站托管服务，可以将仓库中的网页文件自动部署为可访问的网站。<br><br>
**Jekyll** 是 GitHub官方支持的一个静态博客框架，使用过程中无需自己部署，GitHub会自动编译。<br><br>
部署博客网站流程如下：<br><br>
1.创建GitHub仓库。<br>
  在GitHub上创建名为`用户名.github.io`的仓库，GitHub通过该命名规则来生成个人主页。<br>
2.配置 Jekyll 环境。<br>
  该文件用于设置网站标题、描述以及主题。<br>
3.创建博客文章目录与博客文章<br>
  创建文件夹：  `_posts/`   用于存放博客文章。在该目录下创建文件并进行编写。<br>
4.创建主页文件<br>
  创建 `index.md` 文件作为网站首页，使用网格布局展示已发布的文章。至此，访问`Lic233.github.io`即可访问博客网站。<br><br>
  结论分析与体会：<br>
  通过本次实验，掌握了使用 GitHub Pages 和 Jekyll 搭建静态博客的基本方法，理解了静态网站生成的原理及流程。<br>
  同时，本实验为学习前端开发、网站设计以及项目展示提供了良好的基础。通过不断完善博客内容，可以逐步形成个人技术积累与作品展示平台。<br>
